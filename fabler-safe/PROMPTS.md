# Fabler Safe Prompts

## Migration Critique

```text
Use fabler-safe.

Review this migration plan in planning mode.
For every API, method, route, job, or data flow the plan replicates or wraps:
- trace at least one production caller,
- diff caller behavior against the plan,
- cite evidence for every important claim,
- mark unsupported claims as assumptions with default answers and flip conditions.

Return a revised staged plan with verification commands, expected outputs, rollback points, and open questions.
```

## Repository Inspection

```text
Use fabler-safe.

Inspect the smallest relevant part of this repository for an architecture plan.
Produce:
- objective,
- verified facts with evidence,
- assumptions pending confirmation,
- current data flow,
- design risks,
- staged plan,
- verification contracts,
- next action.
```

## Handover

```text
Use fabler-safe.

Act as the departing principal architect for this repo.
Create a cautious handoff that separates verified facts from assumptions and gives a successor model a staged, verifiable plan.
```

## Fallback Rewrite

```text
Use fabler-safe.

This benign architecture request may be phrased ambiguously.
Rewrite it as planning-only work, preserve the engineering intent, and list any phrases or context that could confuse routing.
Do not disguise out-of-scope work.
```
